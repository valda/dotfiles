# 作業進捗作成コマンド

$ARGUMENTS の期間のコミットログから進捗報告を作成して下さい。

- 引数がなければ本日、あるいは直近の一日で。
- git log の取得では `--oneline` を付けずコミットメッセージの詳細まで確認して下さい。ただし `-p` (ソースコードの差分)までは必要ありません。
- 基本的にコミットの時系列でまとめる。
- 些細な修正はまとめて簡潔に、フォーマットはプレーンテキスト形式で。
- コミット規模の大きなものは、変更の粒度や重大性・影響度の大きさ順にまとめる。ただし、レポート中に "最大規模", "最高影響度" のような評価結果の出力は不要。
- 報告中に"完全"という単語を使うのを禁止。"完全実装"や"完全解消"や"完全確立"といった誇大なレポートは行わない。
- 実装の詳細（クラス名やソースファイル名）に立ち入った表現はなるべく控えて、"◯◯の機能に対する□□の追加"というような外形的な表現を心がける。
