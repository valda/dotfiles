## あなたのタスク

$ARGUMENTS の日時以降、あるいは期間のコミットログから進捗報告を作成して下さい。
$ARGUMENTS がなければ本日、あるいは直近の一日から作成してください。

## 進捗報告のガイドライン

- フォーマットはプレーンテキスト形式。
- git log の取得では `--oneline` を付けずコミットメッセージの詳細まで確認する。ただしソースコードの差分まで確認する必要はありません。
- 規模の大きなコミットは、変更の影響度の大きさ順にまとめる。ただし、レポート中に "最高影響度" のような評価結果は出力しない。
- 些細な修正は簡潔にまとめる。
- 上記以外はコミットの時系列でまとめる。
- レポート中に "完全実装" や "完全解消" といった誇大なレポートは行わない。
- 実装の詳細（クラス名やソースファイル名）に立ち入った表現は控える。"◯◯の機能に対する□□の追加"といったような外形的な表現を心がける。
